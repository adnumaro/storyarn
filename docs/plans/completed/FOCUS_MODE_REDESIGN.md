## 1. Motivation

### Problems with Current Layout

1. **Sidebar shows irrelevant content.** The SHEETS tree appears on every screen, even Localization and Assets where it serves no purpose. No visual indicator of which tool is active.

2. **Sidebar wastes space.** Localization (table-heavy) and Assets (grid-heavy) lose ~230px to a sidebar that provides no value. Translators and asset managers need maximum horizontal space.

3. **Inconsistency between screens.** Sheets, Flows, Maps, Screenplays, Localization, and Assets all use different layouts, different topbar contents, and different content structures. This increases cognitive load when switching tools.

4. **No focus mode.** Narrative designers (writers) need distraction-free environments. Current UI always shows chrome. Writing tools (iA Writer, Ulysses, Scrivener) prove that minimalist interfaces enhance creative flow.

5. **Tree doesn't scale.** In large projects (50+ sheets, dozens of flows), expanding/collapsing tree nodes to find content is slow. The Planescape: Torment stress test demonstrated this with 30+ sheets.

6. **Context switching penalty.** Opening block config in sheets requires shifting visual focus from the content area to the sidebar panel. The toolbar+popover pattern (already used in Maps and Flows) keeps focus near the content.

### What We Keep

- All existing functionality â€” nothing is removed
- Tree navigation â€” available as pinnable panel
- Config panels â€” migrated to inline toolbars + popovers
- All actions and shortcuts â€” relocated, not eliminated

## 7. Phase Plan

### Phase 1: Layout Restructure (Low Risk)

**Goal:** Reorganize visual structure. No functionality removed. Tree pinned by default.

- [ ] Create new unified layout component (`Layouts.project_v2` or replace `Layouts.project`)
- [ ] Left floating toolbar (tool icon, trash, back)
- [ ] Right floating toolbar (settings, presence, user)
- [ ] Tree panel as pinnable component (pinned by default)
- [ ] Tool switcher dropdown in tree panel header
- [ ] Project title component (FigJam pattern: collapsed/expanded)
- [ ] Dynamic tree content based on active tool
- [ ] Migrate all 6 tool screens to new layout
- [ ] Remove old sidebar component
- [ ] Responsive: panels fullscreen on mobile with close button

**Outcome:** Same functionality, unified visual language, more space for Localization/Assets.

### Phase 2: Command Palette

**Goal:** Fast keyboard-driven navigation and actions.

- [ ] `Cmd+K` opens palette overlay
- [ ] Fuzzy search across all project entities (sheets, flows, maps, screenplays)
- [ ] `/goto sheets #nameless-one` â€” navigation with autocomplete
- [ ] `/goto flows "Morte Intro"` â€” fuzzy search
- [ ] Global search across content
- [ ] Keyboard shortcut hints in toolbar tooltips ("Cmd+K for quick actions")

### Phase 3: Tabs

**Goal:** Navigate between recently opened entities without tree or palette.

- [ ] Tab bar below project title or above content
- [ ] Click on entity â†’ opens as tab
- [ ] Tabs persist within session
- [ ] Close tabs individually
- [ ] Tab overflow handling (scroll or dropdown)

### Phase 4: Command Palette â€” Extended

## 8. Open Questions

1. **Background/theme for content area.** Dark desk (like draft mockup) vs neutral/adaptive per tool? Canvas tools (flows, maps) already have grid backgrounds. Document tools (sheets, screenplays) need appropriate contrast for form inputs and text editing.

2. **Transition animation.** When switching tools, should the tree panel animate its content change? Fade? Instant swap?

3. **Tab persistence.** Should tabs survive page reload? (LocalStorage vs server-side)

4. **Tree panel default width.** Current sidebar is ~230px. Tree panel could be narrower since it doesn't need the TOOLS section anymore.

5. **Keyboard shortcuts catalog.** Which shortcuts should be global (always work) vs tool-specific?
